## CS-01: 灵感捕捉

**描述**: 用户在任何场景下快速捕捉灵感，不打断当前心流

**用户类型**: 学术矿工、碎片思考者

**核心价值**: 零摩擦记录，灵感不再丢失

**用户旅程**:
```
[灵感闪现] → [打开应用] → [输入内容] → [自动保存] → [回到原事]
```

---

### MS-L-01: 首次捕捉灵感

**复杂故事**: CS-01
**故事类型**: 光明故事 (Light)
**优先级**: P0
**故事点数**: 3

#### 用户故事

作为一名**博士研究生**，
我想要**在阅读论文时快速记录突然冒出的想法**，
以便**不打断阅读心流，同时确保灵感不丢失**。

#### 故事评估（光明故事：从坏到好）

| 维度 | 描述 |
|------|------|
| **起始状态** | 灵感稍纵即逝，经常遗忘重要想法 |
| **转折点** | 应用提供即时输入框，无需任何前置操作 |
| **结束状态** | 灵感已被安全记录，用户安心回到原来的事情 |
| **情感体验** | 从焦虑（怕忘）→ 安心（已记录）→ 轻松（继续工作） |

#### 验收标准

**场景 1: 正常流程**
- Given: 用户打开 ResearchFlash 应用
- When: 应用完成加载
- Then: 首屏直接显示输入框，光标自动聚焦

**场景 2: 输入保存**
- Given: 用户在输入框中输入内容
- When: 用户点击发送按钮或按回车
- Then: 灵感卡片出现在列表中，输入框清空

**场景 3: 字数限制**
- Given: 用户输入内容接近 280 字
- When: 达到 250 字时
- Then: 显示剩余字数提示

#### 用户可感知变化

| 阶段 | 用户感知 |
|------|---------|
| 操作前 | 看到空白输入框 + 闪烁光标 |
| 操作中 | 输入文字，看到字符即时出现 |
| 操作后 | 灵感卡片淡入列表，感受"已保存" |

#### 约束（来自 real.md）

- 首屏必须直接展示输入框
- 单条灵感限制 280 字以内

#### 依赖

- **需要**: 无（入口故事）
- **开启**: MS-L-02, MS-G-01

### MS-L-02: 离线捕捉灵感

**复杂故事**: CS-01
**故事类型**: 光明故事 (Light)
**优先级**: P0
**故事点数**: 3

#### 用户故事

作为一名**在地铁上通勤的知识工作者**，
我想要**在没有网络的情况下也能记录灵感**，
以便**不受网络环境限制，随时捕捉想法**。

#### 故事评估（光明故事：从坏到好）

| 维度 | 描述 |
|------|------|
| **起始状态** | 地铁隧道无网络，担心无法保存 |
| **转折点** | 应用支持离线写入，本地优先存储 |
| **结束状态** | 灵感已保存到本地，网络恢复后自动同步 |
| **情感体验** | 从担忧（无网络）→ 惊喜（可以用）→ 信任（靠谱） |

#### 验收标准

**场景 1: 离线写入**
- Given: 用户处于离线状态
- When: 用户输入并提交灵感
- Then: 灵感保存成功，显示"已保存到本地"

**场景 2: 自动同步**
- Given: 用户有未同步的本地灵感
- When: 网络恢复
- Then: 自动同步到云端，无需用户操作

#### 约束（来自 real.md）

- 必须支持离线写入
- 网络恢复后自动同步

#### 依赖

- **需要**: MS-L-01
- **开启**: 云端备份功能（v1.0）

### MS-D-01: 字数超限提示

**复杂故事**: CS-01
**故事类型**: 黑暗故事 (Dark)
**优先级**: P1
**故事点数**: 2

#### 用户故事

作为一名**输入内容过长的用户**，
我想要**在超出字数限制时得到友好提示**，
以便**知道如何调整内容，而不是突然被截断**。

#### 故事评估（黑暗故事：从好到坏再恢复）

| 维度 | 描述 |
|------|------|
| **起始状态** | 用户正常输入，体验流畅 |
| **转折点** | 内容超过 280 字，触发限制 |
| **结束状态** | 用户收到提示，精简内容后成功提交 |
| **情感体验** | 从流畅 → 受阻（超限）→ 被帮助（提示）→ 恢复 |

#### 验收标准

**场景 1: 即时反馈**
- Given: 用户输入内容
- When: 字数达到 250 字
- Then: 显示"还剩 30 字"提示，颜色变为警示色

**场景 2: 阻止超限**
- Given: 用户输入内容达到 280 字
- When: 用户继续输入
- Then: 阻止输入，提示"已达字数上限"

#### 约束（来自 real.md）

- 单条灵感限制 280 字以内

#### 依赖

- **需要**: MS-L-01
- **开启**: 无

### MS-G-01: 日常捕捉灵感

**复杂故事**: CS-01
**故事类型**: 灰色故事 (Grey)
**优先级**: P1
**故事点数**: 2

#### 用户故事

作为一名**已使用一周的用户**，
我想要**每天都能快速记录多条灵感**，
以便**养成随时捕捉想法的习惯**。

#### 故事评估（灰色故事：循环）

| 维度 | 描述 |
|------|------|
| **循环模式** | 每天多次打开应用 → 输入 → 关闭 |
| **优化点** | 启动速度、输入效率、保存反馈 |
| **习惯形成** | 从刻意使用 → 自然习惯 → 肌肉记忆 |
| **情感体验** | 从费力 → 轻松 → 自动化 |

#### 验收标准

**场景 1: 快速启动**
- Given: 用户从桌面点击应用图标
- When: 应用启动
- Then: 300ms 内显示输入框

**场景 2: 连续输入**
- Given: 用户刚提交一条灵感
- When: 想要继续输入下一条
- Then: 输入框已清空，光标已聚焦，可立即输入

#### 依赖

- **需要**: MS-L-01
- **开启**: 无（持续使用）

## CS-02: 灵感重现

**描述**: 系统定期推送沉睡的旧灵感，帮助用户重新发现价值

**用户类型**: 已有积累的用户

**核心价值**: 被动回顾，惊喜重现

**用户旅程**:
```
[收到推送] → [点击查看] → [浏览旧灵感] → [处理归档] → [感受惊喜]
```

---

### MS-L-03: 首次收到灵感推送

**复杂故事**: CS-02
**故事类型**: 光明故事 (Light)
**优先级**: P0
**故事点数**: 3

#### 用户故事

作为一名**使用一周后的用户**，
我想要**收到系统推送的旧灵感**，
以便**重新发现被遗忘的想法，产生新的连接**。

#### 故事评估（光明故事：从坏到好）

| 维度 | 描述 |
|------|------|
| **起始状态** | 一周前的灵感已被遗忘 |
| **转折点** | 周日晚 8 点收到推送通知 |
| **结束状态** | 重新看到旧灵感，感叹"原来我想过这个" |
| **情感体验** | 从遗忘 → 惊喜（收到推送）→ 感慨（重新发现） |

#### 验收标准

**场景 1: 定时推送**
- Given: 用户有 7 天前创建的灵感
- When: 周日晚 8 点
- Then: 用户收到推送通知"本周灵感回顾"

**场景 2: 查看回顾**
- Given: 用户点击推送通知
- When: 应用打开
- Then: 显示本周待回顾的灵感列表

#### 约束（来自 real.md）

- 每周日晚 8 点推送本周灵感回顾

#### 依赖

- **需要**: MS-L-01, MS-G-01（需要积累灵感）
- **开启**: MS-G-02

### MS-G-02: 每周回顾灵感

**复杂故事**: CS-02
**故事类型**: 灰色故事 (Grey)
**优先级**: P1
**故事点数**: 2

#### 用户故事

作为一名**养成回顾习惯的用户**，
我想要**每周固定时间处理推送的灵感**，
以便**形成定期整理的习惯，保持灵感库清爽**。

#### 故事评估（灰色故事：循环）

| 维度 | 描述 |
|------|------|
| **循环模式** | 每周日晚收到推送 → 浏览 → 归档 → 等待下周 |
| **优化点** | 批量处理效率、归档操作便捷性 |
| **习惯形成** | 从被动提醒 → 主动期待 → 固定仪式 |
| **情感体验** | 从费力 → 轻松 → 期待（周日回顾时刻） |

#### 验收标准

**场景 1: 批量归档**
- Given: 用户在回顾页面
- When: 处理完一条灵感
- Then: 自动显示下一条，无需返回列表

**场景 2: 完成回顾**
- Given: 用户处理完所有待回顾灵感
- When: 最后一条归档完成
- Then: 显示"本周回顾完成"+ 鼓励语

#### 依赖

- **需要**: MS-L-03
- **开启**: 无（持续循环）
